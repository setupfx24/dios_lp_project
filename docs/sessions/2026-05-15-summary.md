## What was delivered

### Backend (`apps/api`)

- `ROUTES_ENABLED` env (`broker | admin | all`) routed via
  `AppModule.register()` dynamic module.
- Schema additions on `auth.users`: `user_type`, `admin_role`,
  `totp_secret_enc`, `totp_verified_at`, `recovery_codes_hash[]`,
  `password_changed_at`, `must_change_password`, `suspended_at`.
- `auth.admin_sessions` table (separate cookie, separate JWT secret).
- `admin.pending_actions` table with `chk_pending_actions_no_self_approval`
  and `chk_pending_actions_no_self_rejection` CHECK constraints.
- Admin module mounted under `/api/v1/admin/*`:
  - `AdminAuthController` (login, 2fa/setup, 2fa/verify-setup, 2fa/verify,
    recovery/use, reauth)
  - `AdminAuthService` — TOTP via `otplib`, QR via `qrcode`, secret
    AES-256-GCM-encrypted at rest, recovery codes Argon2-hashed
  - Guards: `AdminJwtGuard`, `TotpVerifiedGuard`, `AdminRoleGuard`,
    `ReauthGuard`
  - `AuditLogInterceptor` — opens a Drizzle tx, sets it on
    `req.adminCtx.tx`, runs the controller, writes the audit row inside
    the same tx. Atomicity invariant: action + audit succeed or fail
    together.
  - Domain controllers: `ApprovalsController` (4-eyes approve/reject with
    self-approval rejected at app, SQL, and DB CHECK layers),
    `InterventionsController` (wallet-adjust with threshold routing to
    `pending_actions`), `BrokersAdminController`, `AdminUsersController`,
    `AuditQueryController`, `OperationsController`.
- All existing broker endpoints moved to `/api/v1/broker/*`.
- `0001_security.sql` extended for `admin` schema grants + 4-eyes
  CHECK constraints.

### Frontend (`apps/admin`)

- Full replacement of the stub. 11 routes.
- Red/orange theme (HSL primary 16°) — visually distinct from broker
  dashboard.
- Hardening middleware: 15-minute idle timeout, strict CSP, X-Frame-Options
  DENY, Referrer-Policy no-referrer.
